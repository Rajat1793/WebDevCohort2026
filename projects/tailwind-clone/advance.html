ChaiTailwind

A utility-first CSS engine written in vanilla JavaScript. Write chai-p-20 instead of CSS.

DOM Manipulation Pattern Parsing Zero Dependencies

How It Works

1

Scan the DOM

querySelectorAll('*') gets every element. We filter for classes starting with chai-.

2

Parse Class Names

Split by -. chai-p-20 gives prefix p and value 20. Look up p in config map to get padding.

3

Apply Inline Styles

Object.assign(el.style, styles) sets the inline style. No CSS file needed at all.

4

Watch for Changes

A MutationObserver fires when new nodes appear. Dynamically created elements are styled automatically.

The Two Parsing Strategies

DYNAMIC MAP prefix to CSS property
chai-p-20 => padding: 20px
chai-bg-red => backgroundColor: red
chai-fs-18 => fontSize: 18px
chai-bw-2 => borderWidth: 2px
chai-p-10-20 => padding: 10px 20px

Split on -, look up CSS property, add px if numeric.

FIXED MAP exact class to style object
chai-flex => display: flex
chai-justify-between => justifyContent: space-between
chai-items-center => alignItems: center
chai-rounded-full => borderRadius: 9999px
chai-shadow-lg => boxShadow: 0 10px ...

Needed because space-between has a - that would break the dynamic parser.

Spacing

chai-p-{n} = padding | chai-m-{n} = margin | chai-p-8-24 = padding: 8px 24px | chai-px-{n} = horizontal padding | chai-py-{n} = vertical padding

Typography

Special parsing case: chai-text-red gives color: red, but chai-text-center gives text-align: center. The parser checks if the value is a text-align keyword before treating it as a color name.

Reactivity via MutationObserver

How it works: We call observer.observe(document.body, { childList: true, subtree: true }). Whenever JavaScript appends a new node, our callback fires and runs applyStyles() on it. The cards below are styled with chai-* classes but no manual style call is made.
